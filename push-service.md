# Как подключить пуш-уведомления

Для работы уведомлений нужно один раз настроить две вещи: сначала подготовить настройки в панели Remnawave, затем активировать уведомления в приложении.

---

## Шаг 1. Найдите или создайте секрет в панели Remnawave

Откройте файл `.env` вашей панели и найдите параметр `WEBHOOK_SECRET_HEADER`.

**Если он уже заполнен** — просто скопируйте его значение. Оно понадобится на следующем шаге.

> [!WARNING]
> Не меняйте существующий секрет. Если вы уже используете вебхуки, смена секрета сломает все, которые сейчас работают.

**Если он пустой** — придумайте любую случайную строку и вставьте:

```env
WEBHOOK_SECRET_HEADER=мой-секрет-123
```

---

## Шаг 2. Подключите приложение

Откройте приложение и перейдите в **Настройки → Соединение → Пуш-уведомления**.

Вставьте скопированный секрет в соответствующее поле и нажмите кнопку **Подключить**.

После подключения приложение выдаст вам **Webhook URL** — скопируйте его, он нужен для следующего шага.

---

## Шаг 3. Обновите настройки панели Remnawave

Вернитесь в файл `.env` и обновите параметры:

### `WEBHOOK_ENABLED`

Если значение `false` — смените на `true`:

```env
WEBHOOK_ENABLED=true
```

### `WEBHOOK_URL`

Если строка пустая — просто вставьте ссылку из приложения:

```env
WEBHOOK_URL=https://ссылка-из-приложения
```

Если там уже есть другие ссылки — добавьте новую через запятую, не удаляя старые:

```env
WEBHOOK_URL=https://старая-ссылка,https://ссылка-из-приложения
```

---

## Шаг 4. Перезапустите панель

```bash
docker compose restart
```

Готово — уведомления будут приходить на ваше устройство.

---

## 🔒 О безопасности

Все чувствительные данные — включая Webhook URL и секрет — хранятся на вашем устройстве в зашифрованном виде. Никто не может получить к ним доступ без вашего ведома.
